  `internal/adapters/credebl/verifier.go` — `log.Printf("[credebl] ResponseVerified raw: %.3000s", ...)` removed. Only `state` + extracted fields are logged.

- [x] **[SEC] Add `--restart unless-stopped` + health check to docker run in deploy.sh** ✓ 2026-05-16
  `deploy.sh` — added `--restart unless-stopped`, `--health-cmd`, `--health-interval=15s`,
  `--health-timeout=5s`, `--health-retries=3` to `docker run` in `start_container()`.

- [x] **[BUG] Race condition in global translator** ✓ 2026-05-16

  `cmd/server/main.go` — translator is built before `loadTemplates` and passed to `funcMap`.

## P3 — Nice-to-have / long-term

- [x] **[OBS] Add Prometheus + /metrics endpoint** ✓ 2026-05-16
  Implemented in `internal/metrics/metrics.go` (stdlib-only, no external dependencies).
  Counters: `credential_issued_total{dpg,schema,status}`, `verification_requested_total{dpg,schema,status}`.
  Histograms: `adapter_duration_seconds{dpg,op}` with buckets 5ms/25ms/100ms/500ms/2s.
  `/metrics` endpoint in main.go protected with optional API key. 6 unit tests in `metrics_test.go`.

- [x] **[OPS] Migrate sessions and status lists to PostgreSQL** ✓ 2026-05-16
  `internal/storage/pg/` — three backends: `SessionStore` (5 s flush, replay on startup),
  `IssuanceLog` (hash-chain preserved, mutex serializes Append), `StatusListStore` (delegates
  bit ops to in-memory Store + syncs BYTEA to Postgres on each mutation).
  `cmd/server/main.go` — `pg.Open` once; `buildSessionStore` and `wireIssuancePG`
  share the pool. Activated with `VERIFIABLY_DATABASE_URL`.

- [x] **[SEC] Hash chain in issuance log (tamper-evidence)** ✓ 2026-05-16
  `PrevHash string` field in `IssuedCredential`. `chainHashOf()` computes SHA-256 of immutable
  fields (ID, SchemaID, IssuerDpg, OwnerKey, IssuedAt, PrevHash) separated by `\x00`.
  `Append()` links each entry to the previous one. `VerifyChain()` detects corruption.
  Backwards-compatible: entries without PrevHash (pre-chain) are not flagged as corrupt.
  4 unit tests in `log_test.go`.

  `internal/storage/pg/issuance.go` — `json.Marshal(c.SubjectFields)` replaced by `var subjectJSON []byte`.

## Architectural Backlog

- [x] **[ARCH] Horizontal scaling: L7 sticky sessions + Redis** ✓ 2026-05-16
  `internal/storage/redis/` — stdlib-only RESP2 client + write-through `SessionStore`
  with local cache and 5 s flush. Activated with `VERIFIABLY_REDIS_URL` (priority over PG).
  `scripts/gen-caddy.sh` — emits `lb_policy cookie verifiably_session` when Redis is active.
  `deploy/compose/stack/docker-compose.yml` — `verifiably-redis` service (redis:7.4-alpine,
  port 6380, RDB). Validated images: `postgres:16.4-alpine`, `redis:7.4-alpine`.

- [x] **[ARCH] Async model for bulk issuance (job queue)** ✓ 2026-05-16
  `internal/jobs/queue.go` — `Queue` with worker pool, `pending chan *jobTask`, subscriber
  channels (SSE-ready), `bulk_jobs` PostgreSQL table with in-memory fallback.
  `internal/handlers/handlers.go` — `BulkJobQueue *jobs.Queue` field in `H`.
  Endpoints: `POST /api/v1/credentials/issue/bulk/async` (202 + job_id),
  `GET /api/v1/bulk/{jobID}` (status JSON), `GET /api/v1/bulk/{jobID}/events` (SSE stream).
  `cmd/server/main.go` — queue initialized with `pgPool` and `VERIFIABLY_BULK_WORKERS` (default 4).

- [ ] **[ARCH] Push revocation notification to holder**
  Currently the holder does not know their credential was revoked until a verifier
  checks the status list. For government use cases (identity credentials), active
  notification is required. Options: webhook to the wallet, or implement the OpenID4VC Notification
  endpoint (if the wallet supports it).

- [ ] **[ARCH] Resolve Docker socket risk in CREDEBL (upstream)**
  `agent-provisioning` requires `/var/run/docker.sock` to dynamically provision Credo containers.
  This gives effective root privileges on the host. Long-term solution:
  collaborate with the CREDEBL team to migrate to a static multi-tenant agent model
  (one Credo container per organization, not one container per agent). Track in the upstream CREDEBL issue tracker.

- [ ] **[ARCH] Separate presentation layer from orchestration layer**
  `verifiably-go` mixes HTML template server, DPG proxy, status list host,
  and REST API in a single process. For production: the REST API (`/api/v1/*`) should be deployable
  without the HTMX UI, and the UI without exposing the API to the internet. Separate with a flag
  `VERIFIABLY_MODE=ui|api|all`.

- [x] **[ARCH] Full observability with OpenTelemetry** ✓ 2026-05-16
  `internal/tracing/` — stdlib-only package (no external dependencies) with:
  `tracer.go` — `Tracer`, `Span`, `SpanContext`; head-based sampler by trace ID.
  `propagation.go` — W3C Trace-Context: `Extract` (incoming request) + `Inject` (outbound header).
  `exporter.go` — `SlogExporter` (one slog line per span; immediate Loki correlation) +
  `OTLPJSONExporter` (batch HTTP POST to Tempo/Jaeger; no protobuf) + `CombinedExporter`.
  `middleware.go` — root span per HTTP request with method, route, status code.
  `global.go` — `SetGlobal` / `Global` / `Start` (API without explicit *Tracer injection).
  `internal/httpx/client.go` — `tracing.Inject` in `DoJSON` / `DoForm` / `DoRaw` → all
  adapters (CREDEBL, walt.id, Inji, LibreTranslate) propagate `traceparent` automatically.
  `cmd/server/main.go` — `buildTracer()` reads `VERIFIABLY_OTEL_ENDPOINT`,
  `VERIFIABLY_OTEL_SAMPLE_RATE`, `VERIFIABLY_OTEL_SERVICE_NAME`; wraps the mux with
  `tracing.Middleware`; drains pending spans on graceful shutdown (10 s timeout).
